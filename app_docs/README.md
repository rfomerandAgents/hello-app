# Application Documentation

This directory contains documentation generated by the ASW (Agentic Software Workflow) system.

## Structure

```
app_docs/
├── README.md                    # This file
├── agentic_kpis.md              # KPI log for workflow runs
├── agentic_kpis_summary.md      # Auto-generated summary metrics
├── assets/                      # Screenshots and images
├── feature-*.md                 # Feature implementation docs
├── bug-*.md                     # Bug fix documentation
└── chore-*.md                   # Maintenance task docs
```

## Documentation Files

Each workflow run generates documentation with:
- Implementation plan and approach
- Code changes made
- Test results
- Review outcomes
- Screenshots (stored in `assets/`)

## KPI Tracking

The `agentic_kpis.md` file tracks:
- Workflow run dates
- Issue classifications
- Attempt counts
- Plan and diff sizes
- Timestamps

Run `uv run scripts/regenerate_kpi_summary.py` to update the summary.

## Naming Convention

Files follow the pattern: `{type}-{asw-id}-{description}.md`

- `type`: feature, bug, or chore
- `asw-id`: 8-character workflow identifier
- `description`: Brief description (kebab-case)

Example: `feature-789eccc8-user-authentication.md`
